Big Data y Razzmatazz: un estudio sobre sus playlists de Spotify

1. Introducción

Razzmatazz es una sala de las salas de conciertos más importantes de Barcelona y no solo eso, es una de las discotecas de referencia en la ciudad condal. Esto es así gracias a sus características y a todas las temáticas y salas de las que dispone cada día de la semana. De las más destacadas y conocidas son El Dirty, Mandanga, la 1 de Razzmatazz y Fuego. El objetivo de esta investigación es analizar las características de las canciones que conforman las playlist de Spotify (creadas por Razzmatazz) de las cuatro fiestas mencionadas anteriormente. Esto se hace para ver si las playlists cumplen con las espectativas del usuario y usuaria media de Razzmatazz. Esto se ha llevado a cabo con la API de Spotify, Spotipy, se usará el Get Audio Features para recolectar datos de las canciones como son la bailabilidad, energía o tempo entre otros (de las cuatro playlists) y luego comparar estos parámetros para extraer conclusiones.

def query_principal(p, offset, contador):  #Se define una función para crear un documento JSON por cada 100 canciones de cada playlist
        query = sp.playlist_items(p, fields=None, offset=offset, market=None) #Se llama a la query o función
        if query['next'] == None:  #Si la query no encuentra la variable "next" quiere decir que ya ha recogido todas las canciones de la playlist 
            with open(f'api-res/playlist-{p}-{contador}-response.json', 'w', encoding='UTF-8') as f:  #Se abre un archivo JSON 
                json.dump(query, f, ensure_ascii=False, indent=4)  #Se mete la query dentro del json
            pass  
        else:  #Si todavia quedan canciones en la playlist
            with open(f'api-res/playlist-{p}-{contador}-response.json', 'w', encoding='UTF-8') as f:  #Se abre un archivo JSON
                json.dump(query, f, ensure_ascii=False, indent=4)  #Se mete la query dentro del JSON
            contador = contador+1  
            offset = offset+100  #Se suma 100 puntos al offset
            time.sleep(1)  
            query_principal(p, offset, contador)  #Se vuelve a lanzar la función con el offset aumentado para que la respuesta comience 100 canciones después de la última respuesta
    
    for p in playlists:  #Se crea un bucle que recorra la lista de playlist
        query_principal(p, offset, contador) #Se lanza la función

Por último, se extraen las features y se crea el DataFrame

f = open('data.json', encoding="utf8")  #Se abre el JSON con los datos de todas las canciones
data = json.load(f)  #Se carga el JSON 
  
total_data = []  #Se crea una lista para llenarla con los datos
  
for d in tqdm(data): #Se recorre el JSON canción por canción
    features = sp.audio_features(d['id']) #Se utiliza la la id de la canción para extraer los features
    d["features"] = features #Se crea una variable de Dataframe con las features
    track_df = pd.DataFrame.from_dict(d, orient="columns") #Se crea un Dataframe con la información de las canciones del JSON 
    features_data = pd.DataFrame.from_dict(features, orient='columns') #Creamos un Dataframe con las features de cada canción   
    final_track_df = pd.concat([track_df, features_data], axis=1) #Se unen los dos Dataframes en uno
    total_data.append(final_track_df) #Se guarda este Dataframe en la lista de Dataframes
    time.sleep(1)  
total_df = pd.concat(total_data)  #Se unen todos los Dataframes en uno resultante
total_df.to_csv("total_df.csv", index=False) #Se convierte el Dataframe final en csv

3. Representación de datos

Seguidamente, se representarán y comentarán los datos de las features que se han extraído para ver si el planteamiento inicial es correcto.

3.1. Features

Popularidad

La popularidad es la fama, reconocimiento o aceptación que tiene una canción.
Tal y como se ha planteado inicialmente, Fuego es la sala con música más alternativa y menos mainstream. Así mismo, se planteaba que El Dirty y Mandanga iban a estar parejos en este aspecto, pero resulta que La 1 también tiene canciones bastante populares y se sitúa como la segunda playlist con más canciones conocidas. Además, analizando las pistas más populares se ha identificado que de las 5 más conocidas, 2 son de la lista de La 1 y 3 de la de El Dirty.

BPM

El BPM o Bit Por Minuto indica el tempo de la música.
Viendo el gráfico, efectivamente Fuego es la lista con el BPM más bajo de media pero La 1 no es la que lo tiene más algo. Aún así, se puede ver que todos tienen resultados tan similares que de la que más a la que menos tiene hay menos de 5 puntos que las separan.

Speechness y Instrumentalness

El speechness representa la cantidad que letra que tiene una canción mientras que el instrumentalness representa canciones con mucha presencia de instrumentos y poca voz.
Con este estudio de datos, se ha detectado que las canciones con un alto speechness no tienen prácticamente instrumental. Como se había planteado, La 1 tiene un nivel de instrumental alta y Fuego tiene un speechness alto. Por otra parte, Mandanga y Dirty no tienen picos de speechness e instrumentalness, mantienen un tono más neutro.

Además, de las 5 canciones con más speechness 4 pertenecen a la lista de Fuego y referente a la instrumentalness de las 5 canciones con más, 4 pertenecen a la lista de La 1 tal y como ilustran los siguientes gráficos.

Danceability

Esta métrica plasma lo bailable que es una canción.
La opinión general marca que las playlist con mayor danceability iban a ser la de Mandanga y la de El Dirty. Viendo los datos se identificó que las dos listas con promedio más alto en este aspecto son las de La 1 y Fuego.

En el top 5 de canciones más bailables 3 pertenecen a Fuego y 2 a Mandanga. Destacar que todos los promedios son bastante parecidos.

Key

La key de una canción es la característica que marca si tiene un tono más agudo o más grave
La primera suposición era que La 1 y Fuego tendrían notas graves, pero resulta que La 1 no destaca en este aspecto. En cambio, la mitad de las canciones de Fuego tienen notas comprendidas entre Do y Re.
